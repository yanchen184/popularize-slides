個人沙箱用
指令名稱叫 --dangerously-skip-permissions，
但畫面上顯示的是 bypass permissions on ── 同一件事，兩種說法。
怎麼切
三種切法 ── 看你在哪個時機。
啟動前
在指令後面接 flag。
claude --permission-mode plan
claude --permission-mode acceptEdits
claude --dangerously-skip-permissions
對話中
按 Shift + Tab 在四種模式之間循環切。
畫面右下角會顯示目前模式 ── plan / 預設 / accept edits on / bypass permissions on
永久設
在 ~/.claude/settings.json 寫死，下次自動套用。
最常用：Shift+Tab。
想規劃時切 plan、放心動手時切 accept、自己玩切 bypass。
永久設　・　寫進設定檔
每次都 Shift+Tab？
不如寫死在設定檔。
~/.claude/settings.json
{
  "permissions": {
    "defaultMode": "acceptEdits",
    "deny": [
      "Read(**/.env)",
      "Read(**/.env.*)",
      "Write(**/.env)",
      "Write(**/.env.*)"
    ]
  }
}
defaultMode
每次啟動的預設模式。寫 acceptEdits 最平衡 ── 改檔不問，shell 還是會問。
deny
底線。.env 一律禁讀禁寫，AI 永遠看不到你的密碼。
想看資深玩家的完整 settings？第三堂講 hooks 時再展開。
偷懶設定
把長指令變成兩個字。
# ~/.zshrc
alias cc='claude --dangerously-skip-permissions'
alias ca='claude --permission-mode auto-edit'
alias cp='claude --permission-mode plan'
設定一次，永久生效。
